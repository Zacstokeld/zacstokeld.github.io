Project Idea

Overview

I intend to create a mobile App that allows you to view, sort and compare AFL & AFLW statistics from through out history. The app will pull the statistics from a database and will allow you to easily query statistics and pull and sort them as you require. This will be achieved through the use of Python, the SQLite module and Pandas which will allow for self-containment, zero-configuration and serverless application.

Motivation

The AFL as an organisation has long been decades behind its international counterparts in regards to the way it delivers its content. From being forced to listen to Brian Taylor every Friday to the disaster that is the official AFL mobile app, it is extremely challenging for consumers to access the information they want. There is a need for a user-friendly, smarter application especially amongst the large amounts of sports gambling, and the thousands of fantasy football coaches who want quick and easy access to whatever stats with whatever parameters they want.

Description

The main feature of this application would be a search function which would allow users to input filters such as player age, team, statistical range (search for players who avg. < 25 disposals per game and >10 marks per game) or position. This search function will work for totals or per game averages, as well as player and team statistics from either a single season, career, or span of seasons you choose. The information will be displayed in an interactive table where you can tap on a column header that displays a statistic once and it will then sort the searched players or teams by that statistic from highest to lowest, tapping on it a second time will then sort by that statistic from lowest to highest, and a third tap will revert back to its original ordering. Furthermore, tapping on a player’s name will then take you to their individual player pages, which displays all that players recorded stats.

Tools and Technologies

LA Python Shell and a code editor, such as Atom, would be needed in order to write the code. BeeWare is an open-source mobile app creation framework that allows you to write in Python and release the application on multiple platforms and OS’s. Its tools allow you to run python and package projects so your app can work on different devices. It also has a relatively low skill-curve, making it a feasible option for someone without much of a skill base. Specifically, ill be using the BeeWare Briefcase tool, which converts your Python code into a standalone application.

Skills Required

  • Learning more complex coding with Python, as well as using modules such as SQLite, and dataframes like Pandas.
  • Understanding, creating and manipulating a database with queries, and how to simplify the interaction for users
  • Using BeeWare and its various tools, such as Briefcase, Toga and Cricket, in order to convert my code, manipulate native widgets and bug test the GUI
